# JPAPP 手動驗收測試清單 (Manual Test Checklist)

本文件用於每次系統模組化、功能外移或版本更新後的手動回歸測試，確保核心遊戲體驗與音訊機制穩定。

---

## 1. 基本啟動 (Initial Load)
- [ ] 遊戲首頁可正常載入，背景與 UI 顯示正確。
- [ ] 「開始冒險」按鈕正常顯示，點擊可進入下一步。
- [ ] 開啟瀏覽器 Console，不應出現與 App 初始化相關的 Error（如腳本載入失敗）。

## 2. 地圖與系統選單 (Map & System Menu)
- [ ] 可順利由首頁進入地圖畫面。
- [ ] 右上角系統 MENU 按鈕可開啟。
- [ ] 「設定頁」可正常開啟，且選項可操作。
- [ ] 「更新日誌」可開啟，並顯示最新版本資訊。
- [ ] **安全性檢查**：在一般正式環境（非 localhost/dev）下，不應在 MENU 看到開發者專用的開關（如 Simple Audio V2 Toggle）。

## 3. 小助靈圖鑑 (Spirit Codex)
- [ ] 圖鑑（Codex）可正常從選單開啟。
- [ ] 已解鎖的小助靈：應正確顯示圖片、名稱及親密度（Bond）。
- [ ] 未解鎖的小助靈：應維持鎖定狀態顯示（問號或剪影）。
- [ ] **100% 滿絆標誌**：已滿絆的小助靈應顯示特定的視覺標誌。
- [ ] 翻頁、Tab 切換與分類過濾（Filter）功能運作正常。

## 4. 導師對話 (Mentor Dialogue)
- [ ] 觸發對話流程正常（如初次進入關卡、特定劇情）。
- [ ] 文字分頁邏輯正常（不應有破圖或文字溢出）。
- [ ] 「下一句」按鈕或點擊畫面可正常推進對話。
- [ ] 「SKIP」按鈕功能正常，可直接跳過對話。
- [ ] 對話結束後，應能正確回到原流程（如地圖或戰鬥開始）。
- [ ] Console 無導師對話系統相關錯誤。

## 5. 戰鬥基本流程 (Battle Flow)
- [ ] 可順利由地圖進入戰鬥畫面。
- [ ] **TAP 模式**：點擊選項可觸發攻擊，且視覺反饋正常。
- [ ] **Flick 模式**：滑動選項可觸發攻擊，軌跡與命中正常。
- [ ] 普通命中：受擊特效與敵人硬直視覺正常。
- [ ] **Combo Popup**：連擊時畫面上方會出現 Combo 數字與回饋字樣。
- [ ] **傷害數字**：命中時敵人身上彈出的傷害數字（紅色/白色）顯示正常。
- [ ] 敵人攻擊：敵人 ATB 滿後會觸發攻擊演出，玩家 HP 減少正常。
- [ ] 勝利流程：擊敗敵人後會進入結算，無卡死。
- [ ] 失敗流程：玩家 HP 歸零後會顯示敗北畫面。

## 6. 滿絆與 GIRAGIRA 重點驗收 (VFX/SFX Priority)
- [ ] **滿絆特效**：100% 滿絆時，發射出的「紅色彗星」軌跡視覺正常。
- [ ] **滿絆命中**：命中敵人時的爆炸特效正確呈現。
- [ ] **滿絆音效**：命中音效應為 `sfx_hit2.mp3`（清脆爆裂音）。
- [ ] **GIRAGIRA 視覺**：觸發 GIRA 攻擊時，特效視覺正常。
- [ ] **GIRAGIRA 音效**：命中音效應為 `sfx_hit3.mp3`（厚實撞擊音）。
- [ ] **優先順序檢查**：若 GIRAGIRA 與滿絆同時觸發，音效應以 **GIRAGIRA (`sfx_hit3.mp3`)** 為主。

## 7. 音訊基本驗收 (Audio Integrity)
- [ ] 地圖 BGM 循環播放正常。
- [ ] 戰鬥 BGM 正常切換，並在戰鬥結束後停止。
- [ ] SFX（按鈕音、戰鬥音）無延遲或爆音。
- [ ] **不衝突檢查**：導師音訊、TTS 朗讀與 SFX 應能共存，不應互相切斷或造成卡頓。
- [ ] **Fanfare 處理**：勝利結算音效（Fanfare）結束並返回地圖後，不應與地圖 BGM 同時重疊播放。
- [ ] **iOS/iPadOS 恢復**：切換分頁後回到遊戲，可透過手勢（如點擊靜音開關或點擊畫面）恢復音訊播放。

## 8. 結算畫面 (Result Screen)
- [ ] 經驗值（EXP）、金幣（Gold）、評分（Rank）與星等（Stars）顯示正常。
- [ ] 「查看出題紀錄」按鈕可開啟紀錄清單。
- [ ] 「返回地圖」按鈕功能正常。
- [ ] 「Next Stage」按鈕在勝利後延遲出現的機制正常。

## 9. 開發者工具與隱私 (Dev-only Verification)
- [ ] 在 `localhost` 或特定開發 Flag 下，Debug Overlay 與 FPS 工具可見。
- [ ] **安全性檢查**：在一般正式環境下，不顯示 `Simple Audio V2` 或 `Performance Overlay` 的切換按鈕。
- [ ] `audioDebug` 面板僅在指定 Flag 或 URL 參數下才出現。

## 10. 每次提交 (Commit) 前最低驗證命令
在進行 Git Commit 之前，請務必執行以下檢查：
```powershell
# 1. 檢查主程式語法
node --check assets/js/game.js

# 2. 檢查新增或修改的 JS 檔案 (範例)
node --check assets/js/vfx-helpers.js
node --check assets/js/spirit-codex-helpers.js

# 3. 驗證更新日誌 JSON 格式
python -m json.tool assets/data/changelog.json

# 4. 檢查是否有異常的空白或衝突標記
git diff --check

# 5. 確認工作目錄狀態
git status --short
```
