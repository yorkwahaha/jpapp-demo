## 第 1 批:答錯回饋層(學習快贏)— ✅ 已完成(2026-06-12)

> 1A 實作後經 York 回饋改版:結算頁內嵌面板會讓彈窗需捲動、出現 slider bar 影響美觀,
> 改為「結算動畫完成後自動開啟出題本(設定 `autoOpenMistakesAfterResult` 可關,預設開)+
> 出題本一鍵錯題置頂 + 錯題卡顯示你的答案/正解與對比說明」。全對或敗北不自動開啟。

### 1A. 結算頁錯題回顧

- **為什麼:** 趁記憶熱復盤,學習效果好;錯題資料(mistakes log)已存在,純呈現層。
- **範圍:** 勝利結算流程加一區塊/一頁:本關錯題列表(句子、你的答案、正解、grammarTip 一行)。無錯題時不顯示。
- **風險:** 低。結算 UI 相鄰 fanfare 音訊與 tally 流程,不動 timing,只加顯示。
- **主要檔案:** `assets/js/result-display-manager.js`、`index.html`(結算模板)、`assets/css/result-mistakes.css`。
- **驗證:** 全對關卡不顯示;錯 1 題/多題顯示正確;不影響 tally 動畫與獎勵流程。

### 1B. 答錯對比教學

- **為什麼:** 助詞混淆對(は/が、に/で、に/へ…)是最大學習痛點;與 1A 同屬答錯回饋層,UI 觸及面相近,合批製作。
- **範圍:** 建混淆對資料表:`(正解助詞, 誤選助詞, skillId 類型) → 一行對比說明`(包裝成兩隻小助靈說明分工的口吻,符合世界觀)。答錯時優先顯示對比說明,查無對比則回退現有 grammarTip。結算錯題回顧(1A)同步顯示對比說明。
- **風險:** 低-中。顯示層擴充,不動 `checkAnswer` 判定;文案需符合 Selene/小助靈語氣與初學者清晰度(核心優先序第一條)。
- **主要檔案:** 新增 `assets/data/particle-contrasts.v1.json`、`game.js` 答錯回饋顯示處(窄改)、`index.html` 模板。
- **驗證:** 對比說明覆蓋主要混淆對;無對比資料時回退正常;文案過一遍人工審(語氣 + 文法正確性)。

## 第 2 批:學習核心

### 2A. 錯題 SRS 複習關(特訓之間)

- **為什麼:** 學習效果投資報酬率最高的一項。
- **範圍:** 地圖加一個特殊入口(如神殿/特訓之間)。出題池 = 玩家錯題(出題本)+ 低 mastery 技能題,按簡化版間隔重複排序(錯越多次越優先、答對後降權)。獎勵可低於正規關,避免刷分。
- **風險:** 中。出題走既有 `generateQuestionBySkill` / `getChoices` 管線(凍結區)——**只新增上游的「選題策略」,不改生成邏輯本身**。戰鬥流程重用現有 battle loop。
- **資料:** mistakes log、`skillMastery` 已存在;可能需在 progression 加 SRS 權重欄位(新增欄位 + fallback 預設,不動舊欄位)。
- **主要檔案:** `game.js`(選題策略 + 地圖入口 glue,窄改)、`index.html`(地圖入口模板)、可能新增 `assets/js/srs-helpers.js`(純函式,可單測)。
- **驗證:** 無錯題時的空狀態;錯題出現頻率符合權重;通關後 mistakes/mastery 正確更新;`node --check` + 正規關卡迴歸。

## 第 3 批:受眾擴大 + 留存

### 3A. 練習模式(神殿修行,無計時)

- **範圍:** 關卡確認彈窗加「修行模式」選項:無 ATB 計時、不扣 HP(或不會死),同題庫同流程,獎勵減半或不計三星。
- **風險:** 中。觸及 battle loop 入口與 timer(高風險區)——以旗標繞過計時啟動,不改 timer 本體邏輯;結算與紀錄需區分模式,不污染 stageBestRecords。
- **主要檔案:** `game.js`(battle 入口旗標 + timer 啟動 guard,窄改)、`index.html`(確認彈窗)。
- **驗證:** 修行模式不觸發計時攻擊;正規模式完全不受影響(重點迴歸);紀錄不被修行成績覆蓋。

### 3B. 每日修行

- **依賴:** 2A 的 SRS 選題策略(重用其出題池)。
- **範圍:** 每日 5 題(錯題+弱項),完成給小獎勵;首頁或地圖顯示今日狀態。Selene 每日一句陪伴文案。
- **風險:** 低-中。需在 localStorage 記「最後完成日期」(新增欄位 + fallback);日期判斷用本地時區即可。
- **主要檔案:** `game.js` 或新 helper、`index.html`、progression 新欄位。
- **驗證:** 跨日重置;當日重複進入的處理;舊存檔載入不壞。

## 第 5 批:新題型與收集(含音訊高風險區)

### 5A. 聽力出題模式

- **為什麼後置:** 觸及音訊鏈(AGENTS 最高風險區)+ 出題管線,需在前面批次都穩定後動工。
- **範圍:** 新題型:播 TTS 句子(空格處理:讀整句或嗶聲代替),玩家填助詞。先以「既有關卡的變體開關」或獨立練習入口落地,不重排關卡表。
- **風險:** 高。TTS 播放沿用現有 token/state guard 模式;iOS Safari 手勢解鎖鏈路必須走現成路徑;失敗 fallback(無聲時自動轉文字題)必須有。
- **主要檔案:** `game.js`(題型分支,窄改)、`audio-tts.js`(盡量只呼叫不改)、`index.html`。
- **驗證:** iOS Safari 實機;TTS 失敗 fallback;與 BGM/SFX 不互搶;舊題型零迴歸。

### 5B. 小助靈家族羈絆

- **範圍:** 家族定義資料化(に家族、と家族…,對應 spirits 資料);共鳴輪或統計頁顯示家族集合進度;集滿家族給獎勵(稱號/對話/小 buff 擇一,從簡)。
- **風險:** 低-中。資料 + 呈現為主;獎勵若涉 battle buff 則需窄 guard。
- **主要檔案:** `assets/data/spirits.v1.json`(加 family 欄位)、codex 顯示層、`game.js` 獎勵判定(窄改)。
